How the Counter Works

A complete technical explanation of how Fingerprint Detector counts, categorises, and scores fingerprinting activity.

1. The Data Flow

Every number you see in the popup badge and UI follows this path:

inject.js → postMessage → content.js → chrome.runtime.sendMessage → background.js → sendResponse → popup.js

API intercepted → { category, api, returnValue, blocked } → tabData.entries[] → Badge + Popup UI

2. What Gets Counted

Every time a page calls a fingerprinting-related API, inject.js intercepts it and calls the report() function:

function report(category, api, returnValue, blocked) {
  window.postMessage({
    type: '__fp_detect__',
    category,       // e.g. 'canvas', 'webgl', 'audio'
    api,            // e.g. 'canvas.toDataURL', 'navigator.userAgent'
    timestamp: Date.now(),
    stack: getCallerStack(),   // which script called it
    returnValue: summarizeValue(returnValue),
    blocked: !!blocked,         // true if ghost/spoof replaced the value
  }, '*');
}

Each call to report() = +1 to the counter. So the badge number is the total number of intercepted API calls, not the number of trackers or scripts.

3. Two Counters: totalCount vs entries.length

The background service worker tracks each tab's data:

const data = {
  domain: 'www.linkedin.com',
  totalCount: 535,     // Always incremented, never capped
  entries: [...],       // Individual call records, capped at 10,000
  categoryCounts: {     // Per-category totals, always accurate
    ua_platform: 148,
    plugins: 65,
    screen: 24,
    // ...
  }
};
FieldWhat it countsCapped?Used for
totalCountEvery API call receivedNoBadge number, popup header, tweet
entries[]Individual call records with full detailsYes (10,000 max)Category breakdown, return values, stacks
categoryCountsPer-category totalsNoSeverity calculation, category sort order
Why the cap? Some pages make thousands of API calls. Storing full details (stack trace, return value, timestamp) for every call would consume too much memory. After 10,000 entries, totalCount and categoryCounts keep incrementing but individual records stop being stored.

4. The Badge Number

The number on the extension icon in the toolbar is totalCount:

function updateBadge(tabId) {
  const count = data.totalCount;
  const text = count > 999 ? '999+' : String(count);
  chrome.action.setBadgeText({ tabId, text });
  chrome.action.setBadgeBackgroundColor({
    tabId,
    color: SEVERITY_BADGE_BG[severity],  // green, orange, or red
  });
}

The badge colour indicates severity:

ColourLevelMeaning
123LowNormal API usage, likely not fingerprinting
456MediumModerate cross-category probing
999+HighHeavy fingerprinting activity across many categories

5. Severity Calculation

Severity is determined by three factors: total API call count, number of categories probed, and whether suspicious categories are involved.

Suspicious Categories

These are APIs rarely needed for normal page functionality but commonly used for fingerprinting:

['audio', 'fonts', 'webrtc', 'battery', 'connection', 'storage', 'extensions']

Aggressive Combo

Canvas + WebGL + Audio together is a known fingerprinting pattern:

const aggressiveCombo = hasCanvas && hasWebgl && hasAudio;

Thresholds

LevelTriggers (any one of these)
HIGH • 8+ categories AND 100+ calls
• Canvas+WebGL+Audio combo AND 75+ calls
• 3+ suspicious categories AND 50+ calls
• 200+ total calls (any categories)
MEDIUM • 5+ categories AND 30+ calls
• 2+ suspicious categories AND 20+ calls
• Canvas+WebGL+Audio combo AND 15+ calls
• 75+ total calls
LOW Everything else

7. Blocked vs Detected

In the popup, you see two numbers:

"535 API calls detected"  // totalCount
"400 blocked"             // entries where blocked === true

The blocked count only applies in Ghost or Spoof mode. An API call is marked blocked: true when the wrapper returns a fake value instead of the real one:

// Ghost mode example:
if (mode === 'ghost') {
  const fakeResult = GHOST[label]();  // e.g. returns 1920 instead of real screen width
  report(category, label, fakeResult, true);  // blocked = true
  return fakeResult;
}

// Detect mode: real value returned, blocked = false
const result = originalGet.call(this);
report(category, label, result, false);  // blocked = false
return result;
Why aren't all calls blocked? Some APIs are intentionally let through even in Ghost/Spoof mode (marked null in the value maps). For example, audio.createOscillator must work for pages to play sound — only the data extraction methods (getChannelData) are blocked.

8. The Popup Category Counts

Each category card in the popup shows how many entries are in entries[] for that category (not categoryCounts). They're sorted by count descending:

const sortedCategories = Object.keys(grouped).sort(
  (a, b) => grouped[b].length - grouped[a].length
);

Inside each category, entries are deduplicated by API name. If navigator.userAgent is called 50 times, you see one entry with "x50" instead of 50 separate entries. The return value shown is the most recent one.

9. The Tweet / Share Count

The share button composes a message using:

const total = currentData.totalCount;         // e.g. 535
const categories = Object.keys(categoryCounts).length;  // e.g. 10
const blocked = entries.filter(e => e.blocked).length;   // e.g. 400

// Result:
"We detected 535 fingerprinting API calls across 10 categories
 on www.linkedin.com and blocked 400 of them"

10. When Counters Reset

EventWhat happens
Page navigation (same tab)All data for that tab is cleared. Counter resets to 0.
Tab closedTab data deleted from memory.
Click "Clear" in popupTab data deleted. Badge disappears.
Extension reloadAll data lost (stored in service worker memory, not persisted).
Switch modeData is not cleared. The mode takes effect on the next API call. Refresh the page to see full effect.

11. Overflow Behaviour

When entries.length reaches 10,000:

  • totalCount keeps incrementing (badge stays accurate)
  • categoryCounts keeps incrementing (severity stays accurate)
  • New individual records are dropped (popup detail view stops growing)
  • Export JSON includes "entriesCapped": true to flag this
